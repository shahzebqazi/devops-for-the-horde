Mac bootstrap

devops-for-the-horde from zero to ready

This is a home base for taking a Mac from a clean install to something that feels like yours quickly: one master config, room for other repos to plug in, and notes that play nicely with Cursor so you are not the only one keeping the README honest.

Tip: if you use agents, point them at config/master.yaml first.

Map

What lives here

This repository

Master config, the GitHub Pages site, and coordination docs — the place that ties the rest together.

Other repositories

Smaller, focused repos for apps, dotfiles, Nix slices, and experiments — linked from the manifest when you add them.

Why

What it is for

  • Get productive sooner — a straight path from “new machine” to “I can actually work” (tools, defaults, a few checks).
  • Personal by design — domains like computing, audio, coding, 3D, and AI are placeholders you can grow over time.
  • Friendly to forks — optimize for your setup; borrow ideas if something helps.

Workflow

Cursor Agents

  1. config/master.yaml — what belongs in this “horde” and which external repos matter.
  2. Linked repos — each repo owns its own automation; this one links and summarizes.
  3. Cursor — AGENTS.md and .cursor/rules so updates to the README and site can stay in sync without guesswork.
